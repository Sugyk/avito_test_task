### Дополнительные решения
Так как любому сервису важно уметь отсеивать некорректные запросы и уметь давать ответ на незапланированные ошибки, к стандартным ответам каждого ендпоинта я добавил так же ошибки 400 - для невалидного ответа, 500 - для внутренней ошибки сервиса.

### Юнит-тесты запускаются командой 
```bash
make unit
```

### Интеграционные-тестыы запускаются командой
```bash
make integration
```

### Сервис поднимается командой 
```bash
sudo docker compose up
```
